# Skill: Generate Candidate System Parts

Use only after finished artifact, first user action, and first system response are clarified.

## Task

Use the learner's idea and co-planning answers to propose candidate system parts.

## Rules

- These are candidates, not the final trail.
- Each part must be a visible system capability.
- Avoid implementation tasks.
- Generate 5-7 candidate parts.
- Include a short reason for each part.
- Mark every part as `ai-suggested`.
- The learner must select, remove, rename, or reorder them.

## Output

Return `CandidateSystemPartsResult`.
